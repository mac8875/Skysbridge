ONE REQUIRED SUPABASE STEP
1. Open your Supabase project.
2. Open SQL Editor > New query.
3. Copy the complete contents of supabase-schema.sql.
4. Paste it into the SQL Editor and press Run.

AUTHENTICATION SETTINGS
In Supabase open Authentication > URL Configuration and set:
Site URL: https://skysbridge.org
Redirect URLs:
- https://skysbridge.org
- https://www.skysbridge.org

In Authentication > Providers > Email, keep Email enabled.
For testing you may temporarily disable Confirm email, but confirmation is recommended for the public website.

WHAT WORKS AFTER RUNNING THE SQL
- Email registration and login
- Protected member panel
- Private memorial submission
- Approved public memorials loaded from Supabase
- Leave a Memory for Sky, saved privately for review
- Row Level Security policies protecting submissions

APPROVING CONTENT
In Supabase > Table Editor:
- memorials: set approved=true only after review
- memories: set approved=true only after review

DEPLOYMENT
Upload every file in this folder to the root of the GitHub repository connected to Netlify.

SECURITY
The publishable key in config.js is intentionally public and protected by Row Level Security.
Never add a secret key or service_role key to the website.
